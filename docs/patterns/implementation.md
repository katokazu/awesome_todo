# 実装方針

## 基本

- Focus 体験に関わる変更は、関連する Todo / Project / Bucket の導線まで確認します。
- ドメイン制約は UI だけでなく、モデルまたはユースケース層でも守ります。
- 日付や期限表示は散らさず、共通ヘルパーに寄せます。
- DB 直接操作を UI コンポーネントに書かず、Repository / Hook 経由にします。

## 状態管理

- UI の一時状態はコンポーネント内に閉じます。
- 複数画面で共有する状態は Zustand を使います。
- 永続化対象は SQLite を優先し、設定や軽量キャッシュのみ Secure Store / AsyncStorage を使います。
